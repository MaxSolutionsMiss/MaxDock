MaxDock DB67 — stable controller and operational interface standard

Core correction
- Stops loading the DB49, DB50, DB52, DB53, DB54, DB55, DB56, DB64, DB65, and DB66 layout runtimes.
- Removes DB49's inline 124px KPI minimum-height enforcement.
- Replaces the overlapping timed layout patches with one stable DB67 controller.
- Uses only scoped metric observers for newly rendered KPI cards; no page-wide layout observer is used.

Interface changes
- Dashboard, My Appointments, Queue, Reports, and User Management use one 78px KPI standard.
- KPI icon glyphs and text are enlarged while icon backgrounds remain compact.
- KPI cards use rounded borders and top accents instead of detached bottom lines.
- Dashboard and Queue labels sit directly before fields without overlap.
- Dashboard Book Appointment and Block Time sit before Date Range.
- Dashboard, Queue, and Reports keep gear and document controls aligned at the far right.
- My Appointments restores a 10px gap before Appointment List and places Appointment List before Notifications.
- Reports remains full width without the redundant left report-view rail.
- User Management summary KPIs and Search users are harmonized with the operational interface.

Roles and preferences
- Coordinator and Shipping Manager navigation: Appointments, Queue, Reports, Dashboard.
- Site Admin navigation also includes Settings.
- System Admin retains all pages and is the only role that sees the top Location selector.
- Customer and Vendor remain limited to My Appointments.
- Dashboard, Queue, My Appointments, and Reports KPI preferences remain supported.

Release marker

Deployment impact
- Frontend-only.
- No Supabase migration.
- No Edge Function deployment.
